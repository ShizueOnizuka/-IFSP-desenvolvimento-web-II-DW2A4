Javascript (ECMAScript 6)



O Es6 (ECMAScript 6) ou ES2015 é uma das atualizações da linguagem. É comum que os usuários se confundam com os nomes que a linguagem possui, mas o nome oficial é ECMAScript ou abreviado para ES. Tem por objetivo ser uma linguagem melhor para projetos mais complexos, corrigir problemas antigos e facilitar o desenvolvimento de libraries (Em resumo é apenas a última versão da padronização da linguagem JavaScript)

Vantagens
Rapidez de leitura e execução;
Suporte por parte da maioria dos navegadores do mercado;
Sem necessidade de compilação;
Código leve;
Grande comunidade de programação disponível.
Desvantagens
Poucos recursos voltados à segurança da informação;
A utilização do Node.js está mais propensa ao vazamento de memória;
Pode contar brechas para a execução de algumas ações maliciosas ao usuário.

Declaração de variáveis
Var x let x const

A diferença principal entre o var e o let é que enquanto o primeiro tem escopo de função, o segundo possui escopo de bloco; O let conserta o antigo problema causado pelo hoisting fazendo com que a declaração de variáveis funcione da forma esperada pela maioria dos desenvolvedores. O const funciona de forma semelhante. A única diferença é que as variáveis criadas não podem ser reatribuídas. É um erro comum achar que const deixa a variável imutável, mas as propriedades de um objeto, por exemplo, podem ser alteradas.


Parâmetro de funções

Algumas pequenas alterações foram adicionadas em relação a parametrização de funções. Apesar dessas mudanças serem pequenas, elas trazem enormes benefícios.


Default parameters

Os parâmetros de funções tem undefined como valor default. Porém, em alguns casos, pode ser necessário utilizar um outro valor.
Com a versão do JavaScript 5 já é possível fazer isso. No ES6 foi introduzido uma nova forma de se fazer isso. Basta adicionar o valor default na definição do parâmetro desejado.


Rest parameters

Na versão JavaScript podemos utilizar o objeto arguments para pegar todos os parâmetros de uma função:
O arguments porém, apresenta alguns problemas:
1. O objeto parece com um array, mas não é exatamente um.
2. Todos os parâmetros da função são automaticamente atribuídos ao arguments. Não temos uma forma clara de diferenciar os parâmetros.


Programação Funcional
arrow functions

Os arrow functions são um excelente syntax sugar na criação de funções, para a resolução do problema de linguagem que existia o this. Uma das formas mais comuns de resolver esse problema é usando o bind() ou então com self = this. Com as arrow functions isso não é necessário. O this funcionará exatamente da forma esperada.


destructuring

Uma nova forma de declarar variáveis extraindo valores de objetos e arrays é através do destructuring que é muito semelhante ao pattern matching, que é uma das bases das linguagens funcionais.


Orientação a Objetos
Classes

Classes nos dão uma sintaxe amigável que definem o estado e o comportamento de objetos que representam as abstrações que usamos diariamente. constructor
O método constructor tem como tarefa fazer a inicialização da instância. Ele é chamado automaticamente na criação da mesma e garante que ela esteja em um estado válido. getters/setters.
Como getters e setters são muito comuns na utilização de classes, o ES6 veio com um syntax sugar para lidar com os mesmos.
Lembrando que a principal função dos getters/setters é proteger os dados internos das instâncias de um objeto.

Podemos utilizar o conceito de herança com o extends.


Módulos
